# Análisis de logs y trazabilidad de ejecución

Devic incorpora un sistema completo de **registro y trazabilidad** que permite inspeccionar el comportamiento interno de los agentes durante cada ejecución.\
A través de los logs, es posible seguir paso a paso las acciones realizadas, las herramientas utilizadas, los mensajes intercambiados con el modelo y el estado de cada tarea.

Este módulo facilita la **depuración, auditoría y optimización continua** del rendimiento de los agentes, ofreciendo visibilidad total sobre cada proceso.

---

## Acceso a los logs

Desde el historial de ejecuciones del agente, cada entrada incluye opciones para **inspeccionar su registro técnico**.\
Al seleccionarla, se abre el panel **Thread Log**, que muestra en tiempo real todos los eventos generados durante la ejecución.

![Panel principal de logs](/open-console-logs-agent.png)



![Panel principal de logs](/logs-console.png)

El panel de logs está dividido en tres niveles de información:

| Nivel            | Descripción                                                                                   |
| ---------------- | --------------------------------------------------------------------------------------------- |
| **LOG / INFO**   | Mensajes informativos del proceso de ejecución.                                               |
| **DEBUG**        | Detalles de bajo nivel sobre llamadas a herramientas, pasos internos y variables intermedias. |
| **WARN / ERROR** | Incidencias o errores detectados durante el proceso.                                          |

---

## Thread Log

El **Thread Log** registra de forma estructurada la conversación técnica entre el agente, las herramientas y el modelo.\
Cada evento queda documentado con su tipo, momento de ejecución y contenido.

![Vista del Thread Log con eventos](/thread-logs-access.png)



![Vista del Thread Log con eventos](/thread-logs.png)

Entre la información disponible se incluye:

- **Llamadas a herramientas** y argumentos utilizados.
- **Respuestas del modelo (LLM Response)**.
- **Mensajes internos del sistema** (inicio, planificación, finalización).
- **Estados de tareas** y pasos completados.

---

## Detalle de mensajes

Seleccionar un evento concreto abre el panel **Assistant Message Details**, donde se muestran las herramientas llamadas por el agente, junto con el _payload_ y la respuesta generada.

![Detalle de llamada a herramienta](/tool-message-detail.png)

Esta vista permite verificar con precisión qué instrucción ejecutó el agente, qué datos procesó y cómo respondió cada herramienta.\
Es especialmente útil para **auditar flujos complejos o depurar comportamientos inesperados**.

---

## Timeline de eventos

El **Thread Events Timeline** ofrece una representación visual y secuencial de toda la ejecución del agente, mostrando cómo se encadenan las acciones del usuario, el modelo y las herramientas.

![Visualización del timeline de eventos](/flow-view-access.png)



![Visualización del timeline de eventos](/timeline-logs.png)

Cada bloque del timeline representa un paso en el flujo de trabajo:

| Elemento             | Descripción                                                                                          |
| -------------------- | ---------------------------------------------------------------------------------------------------- |
| **User**             | Mensaje o instrucción inicial enviada al agente.                                                     |
| **Planning**         | Etapa de planificación interna antes de ejecutar acciones.                                           |
| **Tool Call**        | Ejecución de una herramienta (por ejemplo, consulta RAG, actualización de tarea, generación de PDF). |
| **Tool Response**    | Resultado devuelto por la herramienta.                                                               |
| **Finish Execution** | Cierre del proceso, donde el agente concluye la tarea.                                               |

---

## Exportación y análisis externo

Los registros pueden **exportarse** para su análisis en herramientas externas o integrarse en sistemas corporativos de auditoría y observabilidad.\
Esto garantiza la **trazabilidad completa** de los agentes y refuerza el control operativo en entornos de producción.

---

<Note>
  El sistema de logs de Devic no solo aporta visibilidad técnica, sino que constituye una herramienta esencial para la mejora continua y la validación del comportamiento de los agentes.
</Note>
