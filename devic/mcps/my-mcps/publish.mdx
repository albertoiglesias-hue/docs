## Descripción general

Después de configurar y validar tus tools, el último paso es **publicar** el MCP para que pueda ser utilizado por agentes, asistentes o entornos externos.

Para hacerlo, haz clic en el botón **Publish** ubicado en la parte superior derecha del panel.

![Botón de publicación](/publish.png)

---

## Visibilidad del MCP

Al seleccionar _Publish_, se abre el cuadro de configuración donde eliges quién tendrá acceso al servidor MCP.

Opciones disponibles:

- **Private — Only accessible by you:**\
  El MCP será accesible solo desde tu entorno o por tus agentes.\
  Ideal para integraciones internas o entornos de prueba.
- **Public — Accessible to everyone:**\
  Permite que el MCP esté disponible para cualquier entorno que soporte el protocolo MCP, como ChatGPT u otras instancias de Devic.\
  Esta opción genera una **URL pública de acceso** al servidor.

  ![Publicación pública](/url-mcp-server.png)

---

## URL generada

Durante la publicación, Devic crea automáticamente una URL bajo el dominio:

`https://mcp.devic.ai/<nombre-del-mcp>`

Esa dirección será el endpoint desde el cual agentes o sistemas externos podrán invocar las tools del MCP.

---

<Card title="Publicación completada" icon="check-circle" horizontal>
  Al finalizar la publicación, recibirás una notificación de éxito y el enlace del MCP aparecerá en la parte superior del panel.
</Card>

![Servidor publicado correctamente](/mcp-published.png)

---

## Después de publicar

Una vez activo, el MCP:

- Se aloja en la infraestructura de Devic.
- Puede ser consumido por agentes y asistentes.
- Puede exponerse a LLMs como chatgpt, claude, cursor, etc.
- Se actualiza automáticamente si modificas las tools o la configuración.
