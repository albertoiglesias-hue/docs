# Built-in Tools

Las **Built-in Tools** son un conjunto de herramientas nativas integradas en Devic que permiten a los agentes, asistentes y MCPs ejecutar tareas comunes sin necesidad de desarrollar integraciones externas.  

Estas herramientas cubren funciones clave como:
- **Automatización de tareas y flujos.**
- **Gestión de documentos, correos y bases de datos.**
- **Análisis de información y generación de reportes.**
- **Interacción con servicios externos (SharePoint, web, email, etc.).**
- **Persistencia de memoria y conocimiento.**

---

## 🎯 Objetivo

El propósito de las Built-in Tools es **reducir la complejidad técnica** de las integraciones y proporcionar **capacidades listas para usar**, asegurando que los usuarios puedan construir soluciones inteligentes rápidamente.

Cada herramienta puede ser invocada desde:
- **Agentes:** para ejecutar acciones automáticas en contextos conversacionales.  
- **Asistentes:** para ofrecer respuestas interactivas y personalizadas.  
- **MCPs:** para exponer APIs, integraciones o servicios internos.

---

## ⚙️ Características principales

- **Disponibilidad inmediata:** todas las Built-in Tools están integradas en la plataforma Devic sin configuración adicional.  
- **Compatibilidad completa:** pueden usarse de forma combinada dentro de flujos de agentes o asistentes.  
- **Escalabilidad:** las herramientas pueden integrarse con datos y procesos externos.  
- **Actualizaciones automáticas:** Devic mantiene las herramientas actualizadas sin necesidad de intervención del usuario.  

---

## 📚 Categorías de herramientas

Las Built-in Tools se agrupan en distintas categorías funcionales:

| Categoría | Descripción |
|------------|-------------|
| **Gestión del conocimiento** | Actualización y consulta de información (Update Knowledge, Search Knowledge). |
| **Comunicación** | Envío y lectura de correos electrónicos. |
| **Documentación** | OCR, generación de PDFs, plantillas de texto, gestión de archivos. |
| **Datos** | Bases de datos, hojas de cálculo y conectores SharePoint. |
| **Automatización** | Creación y control de tareas automáticas. |
| **Memoria** | Acceso al contexto conversacional a corto y medio plazo. |
| **Análisis y visualización** | Generación de gráficos y ejecución de código. |

---

## 🧠 Ejemplo de uso

Un agente puede combinar varias Built-in Tools para automatizar un flujo completo, por ejemplo:
1. **Leer un email entrante** con la tool *Read Emails*.  
2. **Extraer información** del adjunto con *OCR*.  
3. **Actualizar la base de conocimiento** con *Update Knowledge*.  
4. **Generar un informe PDF** con *PDF from HTML*.  
5. **Enviar el resultado** mediante *Send Email*.
