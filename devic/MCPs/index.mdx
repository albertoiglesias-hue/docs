## Qué es un MCP

Un **MCP (Model Context Protocol)** es una herramienta o conector que amplía las capacidades de los modelos de lenguaje, permitiéndoles ejecutar acciones sobre sistemas externos o internos.  
En Devic, los MCPs actúan como **interfaces de integración**, conectando tus agentes y asistentes con APIs, servicios o funciones personalizadas.

Cada MCP puede ser:

- **Mis MCPs:** trasnformas tu API en tu servidor MCP con las tools listas para ser usadas por los modelos.  
- **Importado de terceros:** catálogo de MCPs de herramientas de terceros como Google calendar, Hubspot o Jira.  
- **Built in tools:** catálogo de herramientas nativas que facilitan la vida al desarrollador, como por ejemplo, integración con correo electrónico vía protocolo IMAP, crear un PDF desde un HTML o leer un spreadshet.

## Cómo se utilizan los MCPs

Una vez creados o importados, los MCPs quedan disponibles para todos los agentes y asistentes del entorno.  
Pueden ser invocados mediante lenguaje natural o como parte de un flujo automatizado.
