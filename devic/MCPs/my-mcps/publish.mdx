## Publicar un MCP

Una vez que tus tools estén configuradas y validadas, el último paso es **publicar** tu MCP.  
Esto permite que los agentes, asistentes o entornos externos puedan conectarse y ejecutar las tools definidas.

Para publicar, haz clic en el botón **Publish** situado en la parte superior derecha del panel.

![Botón de publicación](f37c8f66-3882-4a16-94b5-74b8cc93b255.png)

---

## Opciones de visibilidad

Al hacer clic en *Publish*, se abrirá el cuadro de configuración de visibilidad.  
Aquí puedes elegir quién tendrá acceso a tu servidor MCP.

Las opciones disponibles son:

- **Private — Only accessible by you:**  
  El MCP será accesible únicamente desde tu entorno o por tus propios agentes.  
  Ideal para integraciones internas o entornos de desarrollo.

  ![Publicación privada](110f78f5-a95c-4430-94f0-8e6adbf438d1.png)

- **Public — Accessible to everyone:**  
  Permite que el MCP esté disponible para cualquier entorno externo que soporte el protocolo MCP (por ejemplo, ChatGPT u otras instancias de Devic).  
  Esta opción genera una **URL pública de acceso** al servidor.

  ![Publicación pública](c8cf98a8-6659-47ad-b55d-ec32ac51d5f6.png)

---

## URL del servidor

Durante la publicación, Devic genera automáticamente una URL bajo el dominio:

`https://mcp.devic.ai/<nombre-del-mcp>`

Por ejemplo:

`https://mcp.devic.ai/suntropy-solar-service`

Esa URL es el endpoint donde los agentes o sistemas externos podrán conectarse para invocar las tools del MCP.

## Qué ocurre tras la publicación

Una vez publicado, el MCP:

- Queda activo en la infraestructura de Devic.  
- Puede ser consumido por tus agentes o asistentes internos.  
- Puede exponerse a otros entornos mediante el estándar **Model Context Protocol**.  
- Se actualiza automáticamente si realizas cambios en las tools o en la configuración del MCP.
