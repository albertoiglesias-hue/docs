## Menú de tools

Una vez importado o creado un MCP, accederás a su panel de configuración principal.  
Desde aquí puedes definir los parámetros generales del servidor y gestionar las *tools* que lo componen.

![Vista del menú de tools](8c04a9df-877a-422e-99fa-201ddb85c1e5.png)

---

## Configuración inicial del MCP

La sección izquierda contiene la configuración base del MCP.  
Cada campo define cómo el servidor MCP se comunicará con tus servicios o APIs:

- **Name:** nombre del MCP que lo identificará dentro de la plataforma.  
- **Description:** breve descripción del propósito o función del servicio.  
- **Server URL:** endpoint raíz de la API que se utilizará para invocar las herramientas.  
- **Authentication Method:** método de autenticación compatible (Basic, Bearer o JWT).  
- **API Key / Token:** credencial necesaria para autorizar las llamadas al servicio.  
- **Template parameters:** variables dinámicas (por ejemplo, `{tenantToken}`) que pueden usarse dentro de las requests.  

<Card
  title="Guardar cambios"
  icon="floppy-disk"
  horizontal
>
  Una vez actualizados los parámetros del MCP, haz clic en <strong>Save Changes</strong> para registrar la configuración.
</Card>

---

## Tools Configuration

En la parte derecha del panel se encuentra el **menú de tools**, que lista los endpoints disponibles o creados dentro del MCP.

Cada elemento del listado representa una tool que puede ser invocada por agentes o asistentes.  
Para cada tool se muestra:

- **Nombre de la función:** nombre técnico o endpoint (`get_study_counts_by_type`, `get_login_register_metrics`, etc.).  
- **Método HTTP** (GET, POST, PUT, DELETE).  
- **Tipo de cuerpo (body)** o parámetros esperados.  
- **Descripción funcional** que detalla su propósito o datos devueltos.

<Card
  title="Importar o añadir tools"
  icon="wrench"
  horizontal
>
  Puedes importar tools directamente desde la documentación de tu API (botón <strong>Import from documentation</strong>) o crear nuevas tools manualmente con <strong>Add Tool</strong>.
</Card>

![Panel de Tools Configuration](6174130d-277d-4ff6-bc28-f76439b0ace7.png)

---

## Resultado final

Una vez guardados los cambios, tu MCP quedará completamente operativo.  
Todas las tools definidas estarán disponibles para los agentes y asistentes que utilicen este conector.

Cada MCP activo puede:

- Ejecutar peticiones reales contra la API asociada.  
- Ser invocado mediante lenguaje natural por un agente o asistente.  
- Publicarse como endpoint externo accesible desde otros entornos (ChatGPT, servicios SaaS, etc.).
