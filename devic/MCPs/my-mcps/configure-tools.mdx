## Configurar las tools

Una vez creado o importado tu MCP, puedes ajustar el comportamiento de cada **tool** desde el panel de configuración.  
Cada tool representa un endpoint de tu API y define cómo el modelo podrá interactuar con él.

![Vista de configuración de tool](b7445a5c-afc9-4155-bc0d-d518a28f34e2.png)

---

## Editor de tool

Al seleccionar una tool, se abrirá el **editor de configuración**, donde podrás definir los siguientes campos:

- **Function name:** nombre interno de la función.  
- **Description:** explica brevemente la función de la tool (ejemplo: “Recupera los estudios categorizados por tipo en un rango de fechas”).  
- **Parameters:** define los parámetros requeridos por la API (nombre, tipo y descripción).  
- **Endpoint:** ruta del servicio (`/api/analytics/studies/counts-by-type`, por ejemplo).  
- **Method:** método HTTP utilizado (GET, POST, PUT, DELETE).

## Modos de edición

Puedes definir una tool en dos modos:

- **Form:** interfaz visual con campos editables para cada propiedad.  
- **JSON:** vista estructurada del esquema completo, compatible con especificaciones OpenAPI.

El botón **Generate** convierte automáticamente los parámetros del modo formulario en la definición JSON correspondiente.

![Vista en modo JSON](6428b69d-3914-4798-9f40-6990f42b83d2.png)

---

## Propiedades avanzadas

En la parte inferior del panel encontrarás las opciones adicionales de configuración:

- **Query Parameters:** parámetros enviados por URL (`?param=value`).  
- **Path Parameters:** variables incluidas dentro de la ruta (`/api/{id}/data`).  
- **Body Property Key:** propiedad principal del cuerpo JSON.  
- **Custom Headers:** cabeceras personalizadas necesarias para la autenticación o el contexto de ejecución.  
- **Widget Binding:** permite vincular la tool con widgets o SDKs embebidos.

## Validación y testing

Una vez configurada la tool, el MCP quedará actualizado y listo para su uso.  
Puedes probar la ejecución directamente desde los agentes o asistentes, o revisar su funcionamiento en la pestaña **Tool Calls**, donde se registran las invocaciones realizadas.
